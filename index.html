The vary mixin is the most complete, scalable and modular theming solution out there.

Quick start

vary covers every aspect of CSS theming, from using body classes to BEM modifiers with all in between, solving lots of consistency and manteinance isssues. Also, it's compatible with Sass 3.2+ and libsass 3.2+.

But let's see how this works. The most common theming pattern is using a .is-foo body class, where foo is the entity for which the CSS variations are destinated to (e.g. user, admin). Each of these entities will have an different styling.

In order to relate the values with the entities, vary uses Sass maps. Then when you are styling a selector, you can include variations using vary. Withing that scope, you can request values like the primary color. By default, vary will loop through the entities in the map and create a selector per entity, each with the corresponding primary color.

$vary-map: (
  user: (
    'color-primary': blue,
    'border': 1px solid #ff0
  ),
  admin: (
    'color-primary': pink,
    'border': 10px solid black
  )
);

.foo{
	background: red;

	@include vary {
		background: vary-get('color-primary');
		border: vary-get('border');
	}
}
			
.foo{
	background: red;	
}

.is-user .foo{
	background: blue;
	border: 1px solid #ff0;
}

.is-user .foo{
	background: pink;
	border: 10px solid red;
}
